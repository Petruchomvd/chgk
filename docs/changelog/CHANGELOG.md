## [2026-04-30] — Исправления Telegram-бота

### Исправлено

- Тренировочный режим в Telegram-боте:
  - возвращена отправка картинки-раздатки, если у вопроса есть `razdatka_pic`
  - убран показ категории/подкатегории в тексте вопроса
- Команды знаний:
  - `/studies` больше не пересекается с `/study`
  - если бот ждёт тему для `/study`, команда `/studies` теперь открывает список статей, а не запускает генерацию по строке `"/studies"`
  - список `/studies` стал читаемее: показывает заголовки статей и число вопросов в основе
- Генерация учебной статьи через `/study`:
  - бот больше не генерирует статью, если в базе не найдено ни одного вопроса по теме
  - вместо этого возвращается понятное сообщение о том, что тема слишком общая или не найдена

## [2026-04-27] — Telegram-бот для персональной подготовки

### Добавлено

- **Telegram-бот на aiogram 3** для персональной подготовки к турнирам ЧГК.
  - `/train` — тренировка вопросами в режимах: случайные, по категории, по турниру (поиск по названию или ID), очередь повторений (Leitner).
  - `/study <тема>` — генерация обучающей статьи по теме на основе вопросов из БД (через OpenRouter, gpt-4o-mini).
  - `/studies` — список ранее сгенерированных статей.
  - `/stats` — личный прогресс (всего ответов, % правильных, по категориям, по Leitner-коробкам).
  - Авторизация по `CHGK_BOT_OWNER_TG_ID` (whitelist, 1 пользователь).
- **`database/training_db.py`** — отдельная БД `training.db` с таблицами `attempts` и `leitner` (5 коробок: 1d, 3d, 7d, 14d, 30d).
- **`app/training_engine.py`** — UI-agnostic движок тренировки (используется ботом, при будущем переезде на VPS не переписывается).
- **`app/study_engine.py`** — генератор статей: ищет вопросы по теме в БД, передаёт LLM, сохраняет в `studies/<slug>.md`.
- **`bot/`** — модуль бота (states, keyboards, handlers/{common,training,study}, main).
- Запуск: `python -m bot.main`. Реюзает `TG_DIGEST_BOT_TOKEN`.
- План: [docs/plans/2026-04-27-personal-prep-app.md](../plans/2026-04-27-personal-prep-app.md).

### Изменено

- **`requirements.txt`** — добавлен `aiogram>=3.4.0`.

### Заморожено

- Дашборд (`dashboard/`) больше не развивается. Разработка ведётся в боте поверх общего движка `app/`.

## [2026-03-07] — Ручная проверка спорных вопросов

### Добавлено

- **`scripts/review_disputed.py`** — скрипт автоматического отбора спорных вопросов для ручной проверки
  - Система dispute_score: низкая уверенность, близкие confidence, confusion-пары, Логика как primary
  - Фильтры по категории, режиму (all/confused/low_conf), лимиту
  - Генерация Markdown-файлов для удобной проверки
- Выгрузки спорных вопросов в `output/`:
  - `review_disputed.md` — 150 самых спорных вопросов
  - `review_disputed_Логика.md`, `review_disputed_Кино.md`, `review_disputed_Быт.md` — по 50 вопросов

### Анализ

- Ручная проверка 150 вопросов из 3 категорий (qwen/qwen-2.5-72b-instruct, 2774 вопроса)
- Результаты: Логика 78%, Кино 72%, Быт 44%
- Быт — главная проблема: модель ставит категорию по поверхностным объектам, игнорируя контекст
- Подробности: `docs/plans/2026-03-07-disputed-questions-review.md`

## [2026-02-28] — Мульти-провайдер, Джентльменский набор, Git LFS

### Добавлено

- **Мульти-провайдерная архитектура** — поддержка 5 LLM-провайдеров:
  - Ollama (локальный), Groq, OpenAI, Anthropic, Google Gemini
  - Единый интерфейс `classify_question()` для всех провайдеров
  - Автоматическая оценка стоимости API-вызовов
  - Параллельная классификация с `--workers N`
- `**scripts/classify_md.py`** — классификация вопросов из .md файлов
  - Парсинг нумерованных вопросов (`N. текст`)
  - Запись результата в блок-цитату `> **Классификация:** ...`
- **«Джентльменский набор»** — анализ частых ответов ЧГК:
  - `scripts/analyze_answers.py` — NER (natasha) + лемматизация (pymorphy3)
  - Извлечение именованных сущностей: люди, места, организации
  - Частотность ключевых слов и биграмм
  - Предвычисление в `data/gentleman_set/*.json`
- **Страница «Джентльменский набор» в дашборде** — 5 вкладок:
  - Люди (PER), Места (LOC), Организации (ORG), Ключевые слова, Биграммы
  - Горизонтальные bar chart + таблицы + drill-down до вопросов
- **Git LFS** для `.db` файлов (`.gitattributes`)

### Изменено

- `config.py` — `MIN_CONFIDENCE` с 0.3 → 0.4
- `requirements.txt` — добавлены `natasha>=1.6.0`, `pymorphy3>=2.0.0`
- `dashboard/components.py` — добавлен `gentleman_bar_chart()`
- `dashboard/db_queries.py` — добавлен `get_questions_by_ids()`
