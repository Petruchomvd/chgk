# Джентльменский набор — целевой pipeline (v2)

**Дата:** 2026-03-01  
**Статус:** Требует реализации

## 1) Цель

«Джентльменский набор» должен показывать **самые частые ответы ЧГК**, сгруппированные по 6 категориям:

1. Люди  
2. Места  
3. Произведения  
4. Наука  
5. Выражения  
6. Числа

Ключевой смысл: это не просто частые слова, а частые **игровые реалии**, которые полезны в подготовке.

## 2) Уточнение требования (важно)

Да, в «Произведения» должны попадать не только книги, но и:
- картины
- фильмы
- сериалы
- пьесы
- песни/альбомы
- игры

Да, в наборе нужны:
- крылатые выражения/цитаты/пословицы (категория «Выражения»)
- абстрактные научные и культурные понятия (категория «Наука», если это концепт/закон/эффект/принцип/тест)

## 3) Чёткие границы категорий

### Люди
Реальные, мифологические и вымышленные персонажи.  
Примеры: `Пушкин`, `Ной`, `Дон Кихот`, `Нил Армстронг`.

### Места
География, города, страны, сооружения и топонимы.  
Примеры: `Венеция`, `Ватикан`, `Эйфелева башня`, `Семь мостов Кёнигсберга`.

### Произведения
Названия или устойчивые именования произведений культуры.  
Примеры: `Мона Лиза`, `Герника`, `Гамлет`, `День сурка`, `4'33"`.

### Наука
Научные термины, эффекты, принципы, тесты, законы, теоремы, модели.  
Примеры: `Кот Шрёдингера`, `Лента Мёбиуса`, `Принцип Паули`, `Эффект Доплера`.

### Выражения
Крылатые фразы, цитаты, пословицы, устойчивые формулы.  
Примеры: `Поехали`, `Аб ово`, `Пусть едят пирожные`, `Предложение, от которого невозможно отказаться`.

### Числа
Числа, годы, числовые константы и устойчивые числовые ответы.  
Примеры: `42`, `451`, `666`, `1984`, `пи`.

## 4) Что сейчас ломает качество

Текущее состояние даёт шум в топах:
- в «Выражениях» появляются стоп-ответы (`не`, `да`, `ничего`)
- много общих предметов уходит в итог без пользы для «джентльменского набора»
- категория `Другое` поглощает большую часть кандидатов

Итог: визуализация есть, но не выполняется требование «показывать именно частые игровые реалии по 6 нужным группам».

## 5) Целевой pipeline

```text
БД (questions.answer)
  -> Шаг A: Извлечение и нормализация ответов
  -> Шаг B: Фильтрация шума (стоп-ответы, низкоинформативные элементы)
  -> Шаг C: Категоризация в 6 целевых категорий (LLM + правила)
  -> Шаг D: Переклассификация спорных кейсов (2-й проход)
  -> Шаг E: Ранжирование внутри категории (частота + покрытие пакетов)
  -> Шаг F: JSON для дашборда (только 6 категорий)
```

### Шаг A. Кандидаты
- брать ответы из `questions.answer`
- чистить разметку/скобки/ударения/кавычки
- аккуратно делить дуплеты/триплеты

### Шаг B. Антишум
- жёсткий стоп-лист для ответов типа: `да`, `нет`, `не`, `ничего`, `один`
- отсекать мусорные однословные ответы без признаков имени/термина/числа
- сохранять отдельный лог отфильтрованных ответов (для аудита)

### Шаг C. Категоризация
- категория выбирается только из 6 целевых
- для неоднозначных ответов использовать правила приоритета:
  1. Числа
  2. Выражения
  3. Произведения
  4. Люди
  5. Места
  6. Наука
- если не удалось уверенно отнести к 6 категориям, кандидат не попадает в публичный топ

### Шаг D. Перепроверка спорных
- второй LLM-проход только по спорным ответам
- ручной whitelist для «канонических» чгк-реалий (`Мона Лиза`, `42`, `Кот Шрёдингера`, и т.п.)

### Шаг E. Ранжирование
Внутри каждой категории сортировать по:
1. частоте упоминаний в ответах  
2. покрытию по разным пакетам (чтобы не было локальных всплесков)

### Шаг F. Публикация
Финальный JSON для UI содержит только 6 категорий.  
Технические корзины и отфильтрованный шум в UI не показываются.

## 7) Требования к дашборду

- Вкладки: только 6 целевых категорий.
- Нет вкладки `Другое` в основном UX.
- В каждой вкладке отображаются:
  - топ-N
  - частота
  - drill-down на вопросы

## 8) Критерии качества (Definition of Done)

1. В топ-20 категории «Выражения» нет стоп-ответов (`не`, `да`, `ничего`, и т.п.).  
2. В «Произведениях» присутствуют разные типы культурных объектов (книги/фильмы/картины/песни и т.д.).  
3. В «Науке» есть реальные концепты (законы/принципы/эффекты/тесты), а не бытовые предметы.  
4. Минимум 90% финальных показанных элементов проходит ручную проверку как «полезный чгк-маркер».  

## 9) Какие файлы менять при реализации

- `scripts/analyze_answers.py` — антишум и фильтрация кандидатов
- `scripts/categorize_gentleman.py` — строгая таксономия из 6 категорий + 2-й проход
- `dashboard/app.py` — убрать `Другое` из пользовательского режима
- `data/gentleman_set/*.json` — регенерация данных после обновления pipeline

## 10) Команды запуска после доработки

```bash
python scripts/analyze_answers.py --top-n 1500
python scripts/categorize_gentleman.py --provider groq --top 1200 --batch-size 40 --force
streamlit run dashboard/app.py
```
