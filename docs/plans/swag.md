# SWAG — Smart Whitelist-Augmented Gentleman's Set

**Дата:** 2026-03-01
**Статус:** К реализации

## Контекст

«Джентльменский набор» — фича дашборда ЧГК, показывающая самые частые **игровые реалии** по 6 категориям: Люди, Места, Произведения, Наука, Выражения, Числа.

**Текущие проблемы:**
- Из ~500 кандидатов категоризировано только ~188 (62% потеряно)
- Дисбаланс: Люди:99, Места:32, Произведения:50+, Наука:**3**, Выражения:**2**, Числа:4
- Ложные срабатывания: палач/нимб→Люди, коса→Места
- Потерянные легитимные кандидаты: железный занавес(30), австралия(21), днк(13), золотое сечение(14), макбет(13)

**Корневые причины:**
1. `is_valid_assignment()` — одновременно слишком строгий и слишком мягкий
2. Маленькие whitelists (16 произведений, 6 выражений)
3. LLM видит только текст ответа, без контекста вопросов
4. Нет pack diversity — локальные всплески раздувают частоту
5. Нет baseline-метрики для управляемой итерации

## Стратегия: Hybrid (D+E)

Детерминистика для очевидных случаев → LLM с контекстом для оставшихся.

## Спринт 1 — Стабилизация качества

### 1.1 Denylist ложных срабатываний

**Файл:** `scripts/categorize_gentleman.py`

Добавить `CATEGORY_DENYLIST` — ответы, ранее неверно присвоенные:

```python
CATEGORY_DENYLIST = {
    "нимб", "палач", "троянский конь", "дирижёр", "гробовщик", "лев",
    "коса", "муравейник", "фейерверк", "мир", "близнецы",
    "чистилище", "подземная железная дорога", "кубик рубика",
}
```

### 1.2 Курированные whitelists (~80-100 элементов)

**Файл:** `scripts/categorize_gentleman.py`

`CURATED_WHITELIST: dict[str, int]` — канонические ЧГК-ответы с детерминистической категорией. Проверяется **ДО** отправки в LLM.

| Категория | ~Кол-во | Примеры |
|-----------|---------|---------|
| Произведения | ~30 | вавилонская башня, ноев ковчег, машина времени, репка, красная шапочка, день сурка, джоконда, монополия, макбет, отелло, вий, челюсти, мышеловка |
| Люди | ~20 | икар, венера, давид, чехов, барби, феникс, ленин, харон, сфинкс, фигаро, пеле |
| Места | ~15 | австралия, япония, ватикан, плутон, марс, сатурн, европа, троя, пиза, израиль, красное море |
| Наука | ~15 | днк, золотое сечение, большой взрыв, глобальное потепление, болезнь альцгеймера, корабль тесея, рентген |
| Выражения | ~10 | железный занавес, сухой закон, камень ножницы бумага, медовый месяц, охота на ведьм, крестовый поход |

### 1.3 Исправить валидаторы категорий

**Файл:** `scripts/categorize_gentleman.py`

- **Люди:** убрать fallback `_has_capitalized_component()` (пропускает "палач"). Оставить только NER PER hints + whitelist. Расширить `PERSON_DENY_MARKERS` профессиями.
- **Произведения:** расширить `WORK_SINGLE_WORD_WHITELIST` до ~30 (репка, макбет, отелло, монополия, челюсти, вий).
- **Выражения:** расширить `EXPRESSION_ALLOWLIST` до ~15. Ослабить маркер — разрешить 2+ словные ответы без функциональных слов.
- **Наука:** расширить `SCIENCE_MARKERS` — "днк", "сечение", "взрыв", "потепление", "альцгеймер", "затмение", "тесей".
- **Места:** добавить `KNOWN_PLACES` set — австралия, япония, ватикан, плутон, марс и т.д.

### 1.4 Pack diversity

**Файл:** `scripts/analyze_answers.py`

- Добавить `pack_id` в SQL запрос
- Трекать `answer_packs: dict[str, set[int]]` — уникальные паки каждого ответа
- Добавить `pack_counts` в `top_answers.json`

## Спринт 2 — Повышение recall

### 2.1 Контекстная LLM-категоризация

**Файл:** `scripts/categorize_gentleman.py`

Для ~200 неклассифицированных кандидатов — передавать в LLM 2-3 текста вопросов:

```
1. железный занавес
```

## Спринт 4 — Режим "По контексту" (новый)

**Мотивация:** Текущий Gentleman Set строится только по ответам. Но многие ключевые ЧГК-реалии (Хемингуэй, Наполеон, Шекспир, Килиманджаро) появляются в **текстах вопросов и комментариях**, а не в ответах. Нужен отдельный режим, показывающий частые реалии по всему контексту.

**Пример:** Вопрос про "Снега Килиманджаро" — ответ редкий, но Хемингуэй упоминается в комментарии и часто встречается в других вопросах.

### 4.1 NER по всем полям (analyze_answers.py)

Добавить новый режим `--mode context` (по умолчанию `--mode answers` — текущий):

- Загружать из БД: `text`, `answer`, `comment`
- Прогнать natasha PER/LOC/ORG по **трём полям** каждого вопроса
- Дедупликация: одна сущность → один вопрос (не считать повторения внутри вопроса)
- Сохранить результаты в отдельные файлы:
  - `data/gentleman_set/entities_context.json`
  - `data/gentleman_set/keywords_context.json`
  - `data/gentleman_set/meta_context.json`

### 4.2 Категоризация контекстных сущностей (categorize_gentleman.py)

- Тот же pipeline из 6 категорий (Люди, Места, Произведения, Наука, Выражения, Числа)
- Но входные данные — из `entities_context.json` вместо `top_answers.json`
- PER → автоматически в "Люди", LOC → "Места", остальные через LLM + whitelists
- Сохранить в `data/gentleman_set/categorized_context.json`

### 4.3 Dashboard — 4-й режим (app.py)

Добавить в радио-кнопки новый вариант: **"По контексту (вопрос + ответ + комментарий)"**

- Показать те же 6 категорийных вкладок
- Дополнительная колонка: "Поле" (text/answer/comment) — откуда чаще упоминается
- Drill-down на вопросы работает так же

### 4.4 Что переиспользуется

| Компонент | Файл | Переиспользование |
|-----------|------|--------------------|
| Natasha NER pipeline | analyze_answers.py:`extract_entities()` | Полностью, новый SQL запрос |
| Keyword extraction | analyze_answers.py:`extract_keywords()` | Полностью |
| Категоризация | categorize_gentleman.py | Те же whitelists/denylist/LLM |

## Альтернативные архитектуры (для справки)

| Подход | Суть | Вердикт |
|--------|------|---------|
| Embeddings | cosine similarity к анкорам категорий | Перспективно, но много инженерной работы |
| Wikidata | API запросы → тип сущности | Дополнительный сигнал, не основной |
| Multi-model voting | 3+ LLM голосуют | Не решает корневую проблему |
| Fine-tune classifier | Обучить на размеченной выборке | Нужен датасет и цикл дообучения |
| **Hybrid D+E (выбрано)** | Правила + whitelists + LLM с контекстом | Лучший баланс усилий и качества |
