# Анализ тематик вопросов ЧГК

## Цель проекта

Матвей готовится к серьёзному турниру по «Что? Где? Когда?» и хочет понять, какие темы встречаются в вопросах чаще всего. Это позволит сфокусировать подготовку на наиболее частых и слабых темах, вместо того чтобы готовиться «ко всему подряд».

На сайте [gotquestions.online](https://gotquestions.online) (~6 500 пакетов, ~235 000 вопросов) **нет поля темы/категории** — только текст, ответ, комментарий, источник и автор. Поэтому тематику каждого вопроса нужно определить автоматически.

## Подход

### Пайплайн

```
[Парсинг сайта] → [База данных] → [LLM-классификация] → [Аналитика + отчёт]
```

1. **Парсинг** — сбор вопросов с gotquestions.online через HTTP (без Selenium)
2. **Хранение** — SQLite база с нормализованной схемой
3. **Классификация** — двухэтапная классификация через локальную LLM (Ollama)
4. **Аналитика** — SQL-запросы, графики, отчёт с рекомендациями

### Ограничения

- Никаких дополнительных платежей (или минимальные)
- Ресурсы ноутбука: Intel i5-11400H, 16 ГБ RAM, NVIDIA RTX 3050 (4 ГБ VRAM)
- Токены Claude Code для проектирования и валидации

## Парсинг

### Стратегия

Сайт gotquestions.online — Next.js SSR-приложение. Все данные встроены в HTML как JSON внутри `self.__next_f.push()`. Selenium не нужен — достаточно `requests.get()`.

### Параметры

| Параметр | Значение |
|----------|----------|
| Задержка между запросами | 0.5с + jitter 0–0.3с |
| Батч | 50 пакетов, пауза 10с |
| Ретраи | 3 попытки с backoff |
| User-Agent | `ChGK-Analyzer/1.0 (research project)` |
| Возобновляемость | Статус каждого пакета в БД |

### Результат

- 806 из 900 пакетов успешно спарсены
- 37 486 вопросов сохранены
- Время парсинга: ~15 минут

## Классификация

### Метод: двухэтапная классификация

Одноэтапный подход (выбор из 52 подкатегорий сразу) показал низкое качество на локальных моделях. Двухэтапный подход значительно лучше:

**Этап 1 — категория:** LLM выбирает 1–2 из 14 категорий (простая задача).

**Этап 2 — подкатегория:** для каждой выбранной категории LLM выбирает подкатегорию из 3–5 вариантов.

### Промпт-стратегия

- Системный промпт с полной таксономией и правилами
- 8 few-shot примеров на этапе 1
- Ключевой вопрос: «Какие ЗНАНИЯ нужны, чтобы ответить?»
- Антибиасные правила (против смещения к «Кино», «Логике»)
- JSON mode для гарантии формата выхода
- Temperature: 0.1

### Выбор модели

Протестированы 3 модели на фиксированном наборе из 50 вопросов (подробнее — см. [benchmark-comparison.md](benchmark-comparison.md)):

| Модель | Точность | Скорость | Стоимость (37k вопросов) |
|--------|----------|----------|-------------------------|
| Qwen 2.5 7B (Q4_K_M) | 46% | 8.6 с/вопрос | $0 |
| Claude Haiku 4.5 | 72% | ~3 с/вопрос | ~$12–15 |
| **Qwen 2.5 14B (Q4_K_M)** | **88%** | 16.9 с/вопрос | **$0** |

**Выбрана модель: Qwen 2.5 14B** — лучшее качество, бесплатна.

### Оценка времени полной классификации

37 486 вопросов × 17 с/вопрос ≈ **7.4 дня** непрерывной работы.

Скрипт возобновляемый — можно прерывать и продолжать.

## Аналитика (планируется)

### Ключевые вопросы

- ТОП категорий и подкатегорий по частоте (% от общего)
- Тренды тем по месяцам (какие растут/падают)
- Сложность по темам (средняя difficulty пакетов)
- Авторы × темы (кто какие темы пишет)

### Визуализация

- Горизонтальная гистограмма: 14 категорий по частоте
- Stacked bar: подкатегории внутри категорий
- Line chart: тренды тем по времени
- Heatmap: категории × месяцы

### Отчёт для подготовки

Markdown-отчёт с ТОП-10 темами, рекомендациями по подготовке, растущими трендами и слабыми местами.

## Запуск

### Предварительные требования

```bash
# Python 3.10+, Ollama установлен
pip install -r requirements.txt
ollama pull qwen2.5:14b-instruct-q4_K_M
```

### Команды

```bash
# Парсинг (уже выполнен)
python scripts/scrape.py

# Классификация (двухэтапная, Qwen 14B)
python scripts/classify.py --twostage --model qwen2.5:14b-instruct-q4_K_M

# Бенчмарк на 50 вопросах
python scripts/benchmark.py --model qwen2.5:14b-instruct-q4_K_M

# Аналитика
python scripts/analyze.py
```
