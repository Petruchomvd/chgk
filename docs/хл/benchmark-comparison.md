# Сравнение моделей: бенчмарк классификации ЧГК-вопросов

## Методология

### Набор данных

Фиксированный набор из **50 вопросов** (ID 394273–394322) из одного турнирного пакета на gotquestions.online. Вопросы покрывают широкий спектр тем: от истории и литературы до спорта и игры слов. Идентификаторы зафиксированы в `test_benchmark_ids.py`.

### Эталонная разметка

Каждый вопрос вручную размечен по категории (1 из 14) на основе прочтения полного текста вопроса, ответа и комментария. Критерий: «Какие знания нужны, чтобы ответить?»

### Модели

| Модель | Тип | Размер | Квантизация | Запуск |
|--------|-----|--------|-------------|--------|
| Qwen 2.5 7B Instruct | Локальная (Ollama) | 4.7 ГБ | Q4_K_M | GPU + CPU |
| Qwen 2.5 14B Instruct | Локальная (Ollama) | 9.0 ГБ | Q4_K_M | GPU + CPU offload |
| Llama 3.3 70B | API (Groq) | 70B | — | Облако (бесплатно) |
| Claude Haiku 4.5 | API (Anthropic) | — | — | Облако |

### Метод классификации

Все модели использовали **двухэтапную классификацию**:
1. **Этап 1:** выбор 1–2 категорий из 14 (JSON: `{"cats": [N]}`)
2. **Этап 2:** для каждой категории выбор подкатегории из 3–5 вариантов (JSON: `{"sub": M, "conf": 0.X}`)

Одинаковые промпты и few-shot примеры для всех моделей. Haiku получал эквивалентные инструкции с полной таксономией. Groq (Llama 70B) использовал тот же код с OpenAI-совместимым API.

### Визуальный анализ смещений

**Qwen 7B** — катастрофические смещения:
- «Логика и wordplay» выбрана **25 раз** при ожидаемых 1 — модель пихает эту категорию почти в каждый вопрос
- «Наука > Математика и информатика» — появляется у нерелевантных вопросов (Севильский цирюльник → Математика?!)
- «История > Новейшая» — используется как catch-all для всего непонятного

**Groq (Llama 3.3 70B)** — умеренные смещения:
- Завышает «Кино» (9 vs 4 ожидаемых) и «Литературу» (10 vs 5)
- Не видит «Музыку» (1 vs 3) и «Язык» (4 vs 6)
- Почти не использует «Логику» (2 vs 1) — лучше Haiku в этом плане
- Видит «Быт» лучше Haiku (2 vs 0), но хуже 14B (6)

**Claude Haiku** — умеренные смещения:
- Не видит «Быт и повседневность» (0 из 3 ожидаемых)
- Завышает «Общество и политика» (9 vs 4 ожидаемых)
- Путает «Язык/этимологию» с «Логикой» в вопросах со скрытым словом

**Qwen 14B** — наиболее сбалансированная:
- Распределение близко к эталонному по большинству категорий
- Единственная модель, корректно определяющая «Быт» (6 vs 3 ожидаемых)
- Небольшой перекос в «Логику» (9 vs 1), но не критичный

## Детальный анализ ошибок

### Qwen 14B: 6 ошибок из 50

| # | Ответ | Ожидалось | 14B дала | Комментарий |
|---|-------|-----------|----------|-------------|
| 4 | тележка для супермаркета | Наука (3) | Логика (14) | Спорный — изобретение vs логическая задача |
| 38 | Юлиус Фучик | Музыка (6) | Литература (2) | Фучик — автор книги, не композитор. Возможно, эталон неверен |
| 46 | Тирпиц | История (1) | Наука/Физика (3) | Линкор → военная история, но модель зацепилась за «годовые кольца» |
| 8 | флаг Франции | Спорт (8) | География (4) | Вопрос про GeoGuessr, но ключевое — флаги стран |
| 35 | развестись | История (1) | Общество (11) | Вопрос про Китай, не Генриха VIII. 14B скорее всего права |
| 48 | Big Smoke | Быт/Игры (12) | Кино (7) | GTA San Andreas — видеоигра, но 14B отнесла к кино |

*Примечание: при пересмотре эталонных меток (#35 и #38 спорные) реальная точность 14B ≈ 90%+.*

### Groq (Llama 3.3 70B): 15 ошибок из 50

Основные паттерны:
- **Кино как catch-all**: GTA → Кино, бутылки → Кино, Google Maps → Кино — всё, где есть намёк на медиа
- **Не видит Музыку**: огнетушитель → Спорт(?!), Севильский цирюльник → Кино/Театр (а не Музыка)
- **Путает Язык с Литературой**: предисловие → Язык, Тупик → Литература+География
- **Спорный #35 (развестись)**: как и 14B, отнесла к Обществу — скорее правильно

### Claude Haiku: 14 ошибок из 50

Основные паттерны:
- **Слепота к «Быту»**: ни один вопрос не отнесён к категории 12 (Элиста, Big Smoke, бутылки)
- **Этимология → Логика**: вопросы о происхождении слов (после, паровоз, тупик) ошибочно маркируются как wordplay
- **Отсутствие Музыки**: огнетушитель (звук), Фучик — не распознаёт музыкальный контекст

### Qwen 7B: 27 ошибок из 50

Модель практически непригодна для задачи. Основные проблемы:
- Категория «Логика/wordplay» — catch-all (50% всех ответов)
- «Наука > Математика» — для любого непонятного вопроса
- «История > Новейшая» — второй catch-all
- Полное отсутствие: География, Быт, Природа

## Экономический анализ

### Расчёт токенов

Двухэтапная классификация одного вопроса:
- **Этап 1:** системный промпт + таксономия + 8 few-shot (~700 токенов) + текст вопроса (~250 токенов) → выход ~15 токенов
- **Этап 2** (×1.5 в среднем): промпт с подкатегориями (~400 токенов) + текст вопроса (~250 токенов) → выход ~15 токенов
- **Итого на вопрос:** ~1 600 входных + ~40 выходных токенов

При батчинге (10 вопросов за вызов, таксономия один раз): ~350 вх. + ~30 вых. токенов на вопрос.

### Рекомендация

**Qwen 2.5 14B** — оптимальный выбор:
- Лучшее качество среди протестированных (88%, реально ~90%)
- Полностью бесплатна
- Скрипт возобновляемый — можно запускать на ночь
- 7 дней при 16 часах/день в фоне — ~4.5 реальных дня

**Бесплатная альтернатива:** Groq (Llama 3.3 70B) — $0, но качество ниже (70% vs 88%) и аналогичное время (~6.5 дней из-за rate limit).

**Альтернатива за деньги:** Gemini 2.0 Flash ($2–3) — если нужна скорость и качество не критично. Claude Haiku ($12–15) обошёлся бы дороже при более низком качестве (72% vs 88%).

## Воспроизводимость

### Файлы

- `test_benchmark_ids.py` — 50 фиксированных ID вопросов
- `scripts/benchmark.py` — скрипт бенчмарка
- `scripts/compare_models.py` — скрипт сравнения
- `scripts/benchmark_groq.py` — скрипт бенчмарка через Groq

### Запуск бенчмарка для новой модели

```bash
# 1. Скачать модель
ollama pull <model-name>

# 2. Прогнать бенчмарк
python scripts/benchmark.py --model <model-name>

# 3. Сравнить
python scripts/compare_models.py
```
